# libdxfrw DWG/DXF Spec-Conformance Review

Master-resident index for the field-level conformance-review artifacts.
The three planning documents live at `~/.claude/plans/` — read them in
the order the round-3 review specifies, later corrects earlier:

1. `libdxfrw-spec-conformance-2026-07-17.md` — the plan (thesis, scope, phases).
2. `libdxfrw-spec-conformance-2026-07-17-addendum.md` — Part A adjacency
   completeness (revised phase map A.15, ledger schema A.16).
3. `libdxfrw-spec-conformance-2026-07-17-subplans.md` — SP1-SP6 implementation-
   ready decomposition + §B0 corrections + Part C per-slice lifecycle.
4. `libdxfrw-spec-conformance-2026-07-17-round3.md` — **authoritative
   corrections layer, overrides 1-3 wherever they conflict.**

## What lives here

- `BRIEF.md` — the 8 shared-brief rules every unit-agent reads first.
- `BASELINE.md` + `BASELINE.failures` — recorded suite-failure allowlist.
- `findings.json` — the register (schema in plan §8.2). Currently empty (seed).
- `errata.json` — DO-NOT-FIX suppression list (seeded with the pilot's
  3 prose-errata per round-3 M2: MTEXT-4, VPORT-V6, SPLINE BD-vs-RD).
- `units/` — one markdown per audited spec unit (row table + verdicts).
- `sweeps/` — ranked-lead outputs from mechanical differs (S1-S6).
- `ledger_src/<unit>.jsonl` — per-unit ledger cell shards, source of
  `spec_ledger.json` regeneration.
- `findings/<slice-id>.jsonl` — per-slice finding shards.
- `gate_results/<slice-id>.json` — validation-gate outcome, written only
  by `validate_slice.sh`.

## The spec text

Per round-3 R.1 (licensing blocker B1), the ODA spec text and PDF are
copyrighted-restricted and are NEVER committed to this GPL repo.
Reviewers regenerate the text from a developer-local PDF:

```
pdftotext -layout /Users/dli/doc/dwg/dwg.pdf libraries/libdxfrw/spec/dwg_spec.txt
```

Identity check (both baked into `SOURCES.json` and the P0 gate):
line count == 14992 and `sed -n '10471p'` startswith `20.4.75 HATCH`.
`libraries/libdxfrw/spec/.gitignore` blocks both files from ever being
staged.

## Branch and push discipline

All conformance work lives on branch `conformance/spec-review-2026-07`
and pushes per-slice (never batched) to a developer remote. Squashed-
merge to master via `--no-ff` tree-preserving merge is the only path
to `landed/on_master` state — see plan A.13 + subplans Part C.7.
